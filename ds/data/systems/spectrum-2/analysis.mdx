# Adobe Spectrum 2: color & tokens

Spectrum 2 is the biggest palette on the roster — 21 full stepped ramps, 16 steps deep
for every chromatic family (<Fact q="palette.structure" />) — and simultaneously the
system most insistent that you never touch it: every one of the 372 palette tokens is
marked `private: true` in the source. The palette exists to feed the alias layers
(<Fact q="tokens.layers" />), where whole scales are remapped onto concepts
(`accent-color-800` → `{blue-800}`) and purposes get named with state suffixes
(`accent-background-color-default`). Unlike Radix, which documents a role for every
step, S2 keeps step semantics loose and pushes meaning entirely into the alias names.

The generation story has to be told honestly as lineage plus corroboration
(<Fact q="palette.generation" />): Leonardo — Adobe's open, contrast-ratio-targeted
generator — is first-party documented for Spectrum 1's colors, but no S2-labeled
source repeats the claim, and the S2 token repo ships only literal `rgb()` values. The
numbers, however, behave like generator output (<Fact q="contrast.strategy" />):
measured from shipped values, `accent-background-color-default` against white lands at
4.81:1 in light and **4.51:1 in dark** — a hair above WCAG AA's 4.5:1 threshold, which
is what ratio-targeted solving looks like and not what hand-picking looks like. S2
publishes no contrast guarantees of its own; the enforcement stance is inferred and
labeled as such.

Delivery is where S2 is genuinely ahead of the industry (<Fact q="tokens.naming" />).
There is no runtime theme object and no class switching: a build-time style macro
compiles token references into atomic CSS, and color modes resolve through the native
CSS `light-dark()` function driven by the `color-scheme` property — the Provider merely
defaults it to OS preferences (<Fact q="modes.support" />). Each mode-aware token
carries three literal value sets — light, dark, and `wireframe`, S2's replacement for
S1's `darkest` theme — so dark mode is a fully separate resolved palette riding the
same token names. Windows High Contrast is a dedicated forced-colors mapping to OS
system keywords rather than a shipped palette. The cost of all this build-time purity:
everything ships 8-bit sRGB, compiled to hex — zero wide-gamut values across 2,469
tokens (<Fact q="palette.colorspace" />), making S2 the rare flagship system with no P3
story at all.

Two findings for comparisons. Focus is fully tokenized
(<Fact q="focus.construction" />): a shared `focusRing()` helper (2px outline, 2px
offset, gated on `isFocusVisible`) sources `focus-indicator-color` → `{blue-800}`, with
matching `focus-indicator-thickness`/`-gap` tokens of 2px each. And the Button
spot-check (<Fact q="component.color-usage" />) shows the private-palette discipline is
aspirational, not enforced: Button mixes semantic values with raw `gray-*` steps and
builds its premium/genai gradients straight from palette stops — token-clean (zero
literals), layer-impure. Token scope beyond color is wide but has sharp gaps
(<Fact q="tokens.scope" />): 1,010 per-component dimension tokens, yet no motion,
breakpoint, cursor, or z-index tokens anywhere.

A practical note for anyone re-verifying: spectrum.adobe.com still serves Spectrum 1
documentation (its color pages are pinned to colors v6.0.0), and s2.spectrum.adobe.com
is a marketing microsite. The real S2 references are the `spectrum-design-data` repo
and the React Spectrum docs — the facts here cite only those.
