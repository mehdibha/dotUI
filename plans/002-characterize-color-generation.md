## Why this matters

A planned refactor (plan 005) moves dark-mode ramp generation from the www wrapper (which today derives dark by reversing light ramps) into the `@dotui/colors` kernel. That refactor must change `contrast`/`material` dark output (that's the fix) while leaving **all light-mode output and all `oklch`/`tailwind` output byte-identical**. Today's tests assert structure (palette names, step counts, reversal relationships) but pin almost no concrete values — only two kernel snapshots (`oklch` and `material`) exist, and they mix light+dark in one snapshot, so "dark changed, light didn't" cannot be verified mechanically. This plan adds value-level snapshots, split by mode, on both sides of the kernel/www boundary, plus one characterization test that documents the current dark-mode gap so plan 005 can flip it into the fixed assertion.

```ts
// Value-level characterization: pins the exact resolved output per algorithm, split by

      expect(resolved.light).toMatchSnapshot('light')

    it('dark ramps match the pinned values', () => {

)
```

### Step 3: Document the dark-mode gap as a characterization test

In the same www spec file, add (imports for `createTheme` come from `@dotui/colors`, already partially imported at the top — extend the import):

```ts
// CHARACTERIZATION of a known infidelity (see plans/005-kernel-owned-dark-mode.md):
// the wrapper derives dark by reversing the light ramp, but the contrast algorithm's
// kernel-native dark mode solves each step against the dark background. The two
// disagree today; plan 005 makes the wrapper use the kernel-native dark and flips
// this assertion to equality.
it('wrapper dark for `contrast` is the reversal, NOT the kernel-native dark solve', () => {
  const wrapper = resolveColorConfig({ ...DEFAULT_COLOR_CONFIG, algorithm: 'contrast' })
  const native = createTheme({
    algorithm: 'contrast',
    palettes: {
      primary: DEFAULT_COLOR_CONFIG.seeds.accent,
      neutral: DEFAULT_COLOR_CONFIG.seeds.neutral,
    },
    modes: { dark: true },
  }).dark
  expect(native).toBeDefined()
  expect(wrapper.dark.accent?.['500']).not.toBe(native?.scales.primary?.['500'])
})
```

**Verify**: `pnpm vitest run www/src/registry/theme/primitives.spec.ts` → exit 0, the new test passes.
