## Installation

```npm
npx shadcn@latest add @dotui/video-player
```

## Usage

Use a video player to embed video with controls that match your design system.

```tsx
import {
  VideoPlayer,
  VideoPlayerControls,
  VideoPlayerVideo,
} from '@/components/ui/video-player'
```

```tsx
<VideoPlayer>
  <VideoPlayerVideo src="/videos/trailer.mp4" />
  <VideoPlayerControls />
</VideoPlayer>
```

## Anatomy

`VideoPlayerControls` renders the default layout for you — a seek bar above a row with play, volume, time, and fullscreen controls — so most players only need `VideoPlayer`, `VideoPlayerVideo`, and `VideoPlayerControls`. Compose the parts by hand for full control over the layout.

```tsx
import {
  VideoPlayer,
  VideoPlayerControls,
  VideoPlayerFullscreenButton,
  VideoPlayerMuteButton,
  VideoPlayerPlayButton,
  VideoPlayerSeekSlider,
  VideoPlayerSkipButton,
  VideoPlayerTime,
  VideoPlayerVideo,
  VideoPlayerVolume,
  VideoPlayerVolumeSlider,
} from '@/components/ui/video-player'
;<VideoPlayer>
  <VideoPlayerVideo src="/videos/trailer.mp4" />
  <VideoPlayerControls>
    <VideoPlayerSeekSlider />
    <div className="flex items-center gap-1">
      <VideoPlayerSkipButton seconds={-10} />
      <VideoPlayerPlayButton />
      <VideoPlayerSkipButton seconds={10} />
      <VideoPlayerVolume>
        <VideoPlayerMuteButton />
        <VideoPlayerVolumeSlider />
      </VideoPlayerVolume>
      <VideoPlayerTime />
      <VideoPlayerFullscreenButton className="ml-auto" />
    </div>
  </VideoPlayerControls>
</VideoPlayer>
```

## Video

`VideoPlayerVideo` is a plain `video` element wired to the player state, so it accepts every native prop — `poster`, `preload`, `loop`, `muted`, `autoPlay`. Clicking the video toggles playback.

```tsx
<VideoPlayer>
  <VideoPlayerVideo
    src="/videos/trailer.mp4"
    poster="/videos/trailer-poster.jpg"
    preload="metadata"
  />
  <VideoPlayerControls />
</VideoPlayer>
```

## Controls

The seek and volume bars are the design system's own `Slider`, and every button is a `Button`, so the player follows your slider and button styles automatically. While the video plays, overlay controls fade out after a short idle period and come back on pointer movement or keyboard focus.

Since the parts are regular components, anything can sit next to them in the control bar — a playback-rate `Menu`, a captions `ToggleButton`, a cast button. The examples below show the common ones.

## Streaming

Real-world video is usually served as HLS or DASH, not a plain file. `VideoPlayerVideo` is a plain `video` element, which is exactly the surface streaming engines attach to — wire up [hls.js](https://github.com/video-dev/hls.js) (free, Apache-2.0) through the `ref`, with Safari falling back to its native HLS support:

```tsx
const videoRef = useRef<HTMLVideoElement>(null)

useEffect(() => {
  const video = videoRef.current
  if (!video) return
  if (video.canPlayType('application/vnd.apple.mpegurl')) {
    video.src = src
    return
  }
  const hls = new Hls()
  hls.loadSource(src)
  hls.attachMedia(video)
  return () => hls.destroy()
}, [])
```

The same pattern works for [dash.js](https://github.com/Dash-Industry-Forum/dash.js) and [shaka-player](https://github.com/shaka-project/shaka-player) (which also covers DRM).

## Beyond this component

This player covers styled controls over your own video: files, HLS/DASH streams, captions, PiP, casting. If you need ads, analytics, DRM license management, or storyboard thumbnails, reach for a full player framework like [Vidstack](https://vidstack.io) or [media-chrome](https://www.media-chrome.org) (both MIT) — they replace this component rather than compose with it.
