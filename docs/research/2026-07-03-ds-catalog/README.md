# ds.dotui.org scored catalog — 77 design systems, 9 domains

> Status: **DONE** — researched 2026-07-03 by a 30-agent Opus 4.8 workflow (8 category recon agents → 1 curator → 20 scoring agents × 4 systems each, web-verified → 1 calibration pass). Snapshot, not kept current.
> Machine-readable output: [`catalog.json`](catalog.json). Extends the [roster selection](../2026-07-03-ds-roster-selection/README.md) (31 systems, color-dimension only) to a full catalog scored across all domains.

**Recon ≠ citation-standard research.** Scores are selection/prioritization evidence: agents opened docs and repos before scoring (sources listed per system), but claims are strong leads, not published facts. Per-system deep dives re-verify everything before anything ships on the site.

## Rubric

Nine domains, 0–10 each, scored against binding anchors from the earlier color/token recon (Spectrum 2 color = 10, shadcn/ui color = 3, USWDS accessibility = 10, Linear openness = 3, Airbnb DLS openness = 0, Vercel Geist docs = 9):

- **color** — color & token system depth: 10 = contrast-solved generation + multi-tier architecture; 3 = static borrowed palette.
- **typography** — tokenized type system, roles, fluid scales, own typeface engineering.
- **spacing** — spacing/sizing/layout/density systematization.
- **components** — breadth × quality; pure token/color systems deliberately score low here.
- **motion** — duration/easing tokens through full choreography language.
- **icons** — own comprehensive icon system vs third-party default.
- **accessibility** — engineering, not claims: contrast-by-construction scores 10, marketing copy scores 2.
- **docs** — public docs quality, rationale, stable URLs.
- **openness** — 10 = source + design data public · 5 = docs-only · 3 = closed but shipped-CSS extractable · 0 = closed.

**general** (0–100) is holistic "how excellent / how much is it worth studying" — explicitly *not* an average: a brilliant narrow system (React Aria, Radix) can outrank a broad mediocre one. 85+ industry-defining · 65–84 excellent · 45–64 solid, specific lessons · 25–44 notable but limited · <25 catalog completeness.

## Scorecards (one line of evidence each)

- **Adobe Spectrum 2** (92) — Contrast-solved Leonardo color with true 3-tier tokens; dimension/component-height/font tokens across desktop+mobile platform scales. React Aria + Spectrum Web Components + CSS give 50+ production components with reference-grade engineered a11y; 959 workflow icons; tokens shipped as JSON-Schema-validated design data plus MCP server. [1](https://s2.spectrum.adobe.com) · [2](https://spectrum.adobe.com/page/design-tokens/) · [3](https://spectrum.adobe.com/page/motion/) · [4](https://github.com/adobe/spectrum-tokens) · [5](https://github.com/adobe/react-spectrum) · [6](https://react-spectrum.adobe.com/icons)
- **Google Material 3** (90) — HCT runtime dynamic color plus complete type-scale-token roles (display/headline/title/body/label) and 4dp/density layout. Spring-based Expressive + Standard motion schemes with duration/easing tokens and choreography is the strongest motion language scored; Material Symbols variable font (2500+ glyphs, 4 axes, 3 styles) is the icon gold standard. 30+ components across Compose/Web/Flutter; theme builder and color-utilities open in 6 ports. [1](https://m3.material.io/styles/motion/easing-and-duration/tokens-specs) · [2](https://m3.material.io/components) · [3](https://m3.material.io/styles/icons) · [4](https://developers.google.com/fonts/docs/material_symbols) · [5](https://github.com/material-foundation/material-theme-builder) · [6](https://supercharge.design/blog/material-3-expressive)
- **IBM Carbon** (90) — Industry-defining open system: ~235 semantic tokens over --cds-* with 4 built-in themes, and dedicated packages for every foundation — @carbon/type (IBM Plex + tokenized scale), @carbon/layout (spacing scale), @carbon/motion (productive/expressive with standard/entrance/exit easing + duration tokens), @carbon/grid, @carbon/icons (2000+ glyph library). 50+ production React components with automated a11y checking (achecker) and exemplary public docs. [1](https://github.com/carbon-design-system/carbon) · [2](https://carbondesignsystem.com/elements/motion/overview/) · [3](https://www.npmjs.com/package/@carbon/motion)
- **Microsoft Fluent 2** (86) — Open Lab/LCH curve-based ramp generator powers a public Theme Designer with a global->alias split (~180 alias color tokens); tokens cover spacing, type, radius, stroke, and motion (durations + four named curves + choreography guidance). Own Segoe UI Variable typeface with platform type ramps, a large open Fluent icon system (Regular/Filled), and 50+ components across Web/iOS/Android/Windows. Industry-defining breadth; a11y is documented WCAG practice rather than by-construction. [1](https://fluent2.microsoft.design/) · [2](https://fluent2.microsoft.design/motion) · [3](https://fluent2.microsoft.design/typography) · [4](https://fluent2.microsoft.design/iconography) · [5](https://react.fluentui.dev/?path=/docs/theme-tokens--docs)
- **GitHub Primer** (84) — Three-tier Style Dictionary pipeline (base->functional->component) with an HSLuv Prism authoring tool and automated WCAG audits over ~100 token pairs per theme, plus an org.primer.llm machine-readable extension - accessibility as engineering, not claims. Three-tier typography tokens, Octicons (600+ icons, multi-size 12/16/24/48px), and 89 components (React + Rails). Motion tokens exist but are the thinnest layer. [1](https://primer.style/) · [2](https://primer.style/product/components/) · [3](https://primer.style/foundations/primitives/typography) · [4](https://primer.style/octicons/)
- **Ant Design** (82) — Three-tier seed->map->alias token system spans color (parametric HSV ramps), typography, spacing (sizeStep/sizeUnit), motion (motionDuration/motionEase), radius, shadow and breakpoints - one of the most complete algorithmic non-color token architectures. ~72 MIT-licensed components with own @ant-design/icons set; motion is principle-led with a companion Ant Motion library rather than deep choreography tokens. A11y is guidance-level, not contrast-by-construction. [1](https://ant.design/docs/spec/introduce) · [2](https://ant.design/components/overview) · [3](https://ant.design/docs/spec/motion) · [4](https://ant.design/docs/react/customize-theme)
- **Twilio Paste** (82) — 100+ components on a two-tier token system (aliases -> semantic) with an explicit Theme + Customization provider layer, Figma variable modes, and full box-shadow/radii/spacing/z-index/font token categories. Exemplary accessibility, inclusive-design and content docs; first-party icon set with guidelines; motion is the weak spot (limited/lightly-surfaced animation tokens). Fully open source with public Figma and token packages. [1](https://paste.twilio.design/) · [2](https://paste.twilio.design/tokens/) · [3](https://paste.twilio.design/tokens/list) · [4](https://github.com/twilio-labs/paste)
- **React Aria Components** (82) — 50+ fully headless components with the deepest a11y engineering of any primitive set — W3C ARIA APG, 30+ locales, 13 calendar and 5 numbering systems, RTL, exhaustive screen-reader testing — plus first-class entry/exit animation hooks (data-entering/exiting, isExiting, Motion integration). Intentionally style-free: no color/type/spacing/icons of its own (that is Spectrum's layer), so those domains score near zero by design. [1](https://react-aria.adobe.com/) · [2](https://react-spectrum.adobe.com/react-aria/styling.html)
- **Cloudscape Design System** (80) — AWS's open-sourced console system: real Sass/JS/JSON design-token package with CTI naming, themeable tokens, comfortable/compact density, and light/dark visual modes; full Open Sans type scale (heading/body/code/display roles), a complete motion language (3 duration tokens + 3 named easing curves + scale/fade/slide choreography + reduced-motion), 130+ tested/accessible components including charts, and extensive pattern guidelines. Exemplary public docs. [1](https://cloudscape.design/foundation/visual-foundation/design-tokens/) · [2](https://cloudscape.design/foundation/visual-foundation/typography/) · [3](https://cloudscape.design/foundation/visual-foundation/motion/) · [4](https://cloudscape.design/components/)

Motion is left to CSS/Framer with no token language; Radix Icons is a small first-party 15x15 set; MIT source public but the color generator is not. [1](https://www.radix-ui.com/themes/docs/theme/typography) · [2](https://www.radix-ui.com/themes/docs/theme/spacing) · [3](https://www.radix-ui.com/themes/docs/overview/getting-started) · [4](https://www.radix-ui.com/primitives/docs/overview/accessibility) · [5](https://www.radix-ui.com/colors)

Own 400+ commerce-focused icon library, purpose-built UI type scale with tabular numerics, and exemplary docs with strong design rationale. React implementation is now deprecated in favor of open web components (maintenance mode), but everything stays public and extractable. [1](https://github.com/facebook/astryx) · [2](https://astryx.atmeta.com/) · [3](https://astryx.atmeta.com/blog/how-astryx-works) · [4](https://www.marktechpost.com/2026/06/27/metas-astryx-brings-a-cli-and-mcp-server-to-an-open-source-react-design-system-agents-can-read/)

## Calibration adjustments

The cross-batch consistency pass changed 3 scores:

- `toss-tds` scores.motion → 5: Toss motion is described as documented motion tokens (instant/fast/standard/slow/page + named easings, prefers-reduced-motion collapse) - equivalent evidence to systems scored 5 (Polaris, Salesforce). - `govuk` scores.color → 6: GOV.UK has a real named Sass palette with govuk-colour/govuk-functional-colour semantic functions (a genuine semantic layer) but no generation - equivalent to NHS (5, static curated + semantic) yet its note describes MORE structure (functional-colour functions). Inconsistent with peers like govbr-ds (6) and gc-design-system (7) that also have semantic layers without generation; 6 aligns curated+real-semantic-layer against the 5-6 anchor band.
- `apple-hig` scores.openness → 5: Consistency check confirms 5 is correct for docs-only (matches Geist=5, LINE=5, Toss=5); no change intended - removing.
