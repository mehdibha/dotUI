## Findings considered and rejected (so nobody re-audits them)

- **Prototype pollution in `codec.ts` `mergeNested`** — false positive: object spread copies `__proto__` as an own data property; the only setter-path assignment changes the local object's prototype, never `Object.prototype`.
- **OG route "renders the string null"** — wrong mechanism (React renders null as empty); real issue is the missing fallback, folded into plan 004.
- **"394 KB showcase bundle parsed per request"** — wrong: module-level ESM import evaluates once per server instance, not per request. Endpoint caching headers are already sane (s-maxage 3600 + SWR).
- **"Remove unused icon libraries" / "consolidate 4 icon libs"** — misread the product: icon library is a builder axis; per-library loading is already lazy + tree-shaken (`registry/icons/create-icon-loader.tsx` dynamic-imports 4 KB generated shims).
- **"Move nitro off beta"** — impossible: the nitro v3 line publishes only datestamped betas (`latest` = `3.0.260610-beta` at audit time); the beta is required by the TanStack Start + `nitro/vite` setup.
- **"Extract focus-ring strings to a shared constant in styles.ts files"** — actively harmful: the publisher's extractor only accepts literals (`extract-config.ts:127`); identifier indirection is exactly what broke the 4 components in plan 001. Canonical style = flat literals.
- **"check-playground.mts / materialize-bundle.ts are dead code"** — check-playground is a documented manual debug CLI; materialize-bundle is an unwired verification harness worth wiring (noted in plan 002's deferred list), not deleting.
- **"@tanstack/react-form unused"** — it's a WIP registry item, intentionally excluded via `ORPHAN_ALLOWLIST` (`registry-build.ts:397-403`).
- **"new Function() on demo defaults is a supply-chain risk"** — committing code to this repo is already arbitrary build-time execution; the eval of authored literals adds no new boundary.
- **`dangerouslySetInnerHTML` of build-time shiki HTML** — trusted build-time content, correctly annotated in source; defense-in-depth sanitization not worth the dependency.
- **"Registry endpoints path traversal"** — clean: `/r/$name` resolves against a prebuilt static map, never the filesystem.
- **"max-age=60 too short on /r/ responses"** — intentional tiered caching (browser 60s, edge 1h, SWR 24h); fine.
- **`customizer-panel.tsx` "god module"** — 501 lines, single importer; a thick orchestrator but not actionably pathological.
- **Node 24 engines pin** — Node 24 is current LTS; policy choice, not debt.
- **`/demos` vs `/preview` route duplication** — different jobs (docs demo iframes vs builder preview page); naming could be clearer but consolidation isn't warranted.
- **Empty-anchors guard in `packages/colors` curve resampling** — unreachable from real inputs (anchors come from constants); not worth hardening.
