## Why this matters

`fumadocs-core@16.8.11` pulls `next@16.0.8` as a transitive dependency (verified: `pnpm why next` → `next@16.0.8 ← fumadocs-core@16.8.11 / fumadocs-mdx@15.0.5`). next 16.0.8 carries three HIGH advisories (DoS via Server Components, an OpenSSL-related infinite loop, RSC cache poisoning — patched in next 16.2.x). dotUI never executes next — it's a TanStack Start app; next is only there because fumadocs supports it — so exploitability is indirect/low, but the advisories drown `pnpm audit` signal and the fix is an in-range minor/patch bump of fumadocs. Newer fumadocs releases within the existing caret ranges require a patched next (≥16.2.7).
